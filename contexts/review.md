# Review Mode

コードの品質・安全性・保守性を重視する。

## 動作指針

- 変更を加える前に既存コードを丁寧に読む
- 問題を指摘するだけでなく、修正案を必ず添える
- 重大度で優先順位を付ける（critical > high > medium > low）
- セキュリティの観点（インジェクション・認証・秘密情報の露出）を必ずチェックする

## レビューチェックリスト

- [ ] ロジックの誤り・エッジケース
- [ ] エラーハンドリング
- [ ] セキュリティ（injection, auth, secrets）
- [ ] パフォーマンス上の懸念
- [ ] 可読性・命名
- [ ] テストカバレッジ
