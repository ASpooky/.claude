# .claude

個人用 Claude Code グローバル設定リポジトリ。

## エージェントハーネスの6コンポーネント

論文定義と Claude Code 等の開発現場での対応をまとめたもの。

| # | コンポーネント | 略称 | 論文での定義（役割） | 開発現場での具体例 |
|---|---|---|---|---|
| 1 | Execution Loop | E | Observe-Think-Act（観察・思考・行動）のサイクル管理。終了条件の判定や、エラーからの自動復帰を制御する。 | 自律的な再試行、エラー時の自動デバッグループ。 |
| 2 | Tool Registry | T | ツール群のカタログ管理。呼び出しのルーティング、監視、入力パラメータのスキーマ検証を行う。 | `.claude/skills/` 配下のカスタムスキルや、標準コマンド群の定義。 |
| 3 | Context Manager | C | プロンプトの文脈を管理し、限られたトークン内で何を LLM に渡すか（Scope negotiation）を最適化する。 | `CLAUDE.md` によるアーキテクチャ規約の付与や、ファイルの読み込み制御。 |
| 4 | State Store | S | エージェントの「記憶」や「状態」を保持するメモリ層。 | `.claude.json` 等に保存されるセッション状態やローカルインデックス。 |
| 5 | Lifecycle Hooks | L | ツール実行の前後やループの節目に、特定の処理や制約を差し込む（フックする）ための仕組み。 | 危険なコマンド実行前の「承認 (y/N) プロンプト」による割り込み。 |
| 6 | Evaluation Interface | V | エージェントの動作結果や生成物が、意図した要件を満たしているかを検証（Verification）する。 | コード変更後に自動で `npm run test` や lint を走らせる検証フロー。 |
